# Shot-by-Shot Script

This script is designed for a short proof-first demo of `system-prompt-benchmark`.

Structure:

`result -> actions -> result`

Target runtime:

- `22-28 seconds`

## Voiceover Script

### Shot 1

Duration:

- `0:00-0:04`

Visual:

- Open on a finished benchmark result in the app.
- Keep the score hero, pass rate, and review queue visible.

Voiceover:

`This benchmark red-teams your system prompt and shows exactly where it breaks.`

### Shot 2

Duration:

- `0:04-0:07`

Visual:

- Hold on the same finished result for a beat.
- Let the viewer register that the app is real and already working.

Voiceover:

`You get a visible score, category signals, and concrete failures to inspect.`

### Shot 3

Duration:

- `0:07-0:11`

Visual:

- Cut back to the workspace controls.
- Show the sidebar with an example prompt, provider, and test-count settings.

Voiceover:

`Start with a prompt, choose a model, and run a focused attack pack or a full benchmark.`

### Shot 4

Duration:

- `0:11-0:15`

Visual:

- Show the benchmark action.
- If possible, include the click on `Start Benchmark`.

Voiceover:

`The app runs adversarial scenarios against the prompt you actually want to ship.`

### Shot 5

Duration:

- `0:15-0:19`

Visual:

- Show one or more product actions after the first result:
- `Analyze Prompt`
- `Build Pack`
- `Compare Versions`

Voiceover:

`Then you can analyze prompt structure, build custom test packs, and compare runs after changes.`

### Shot 6

Duration:

- `0:19-0:24`

Visual:

- Return to a finished result state after the actions.
- Prefer a clearer or improved final frame than the opening frame.

Voiceover:

`So instead of guessing, you can measure regressions, inspect failures, and harden the prompt before deployment.`

## Editing Notes

- Open on proof, not setup.
- Keep cursor motion deliberate.
- Use hard cuts instead of long transitions.
- Do not show installation, terminal setup, or provider-key entry for long.
- If the benchmark run itself takes too long, cut from action to finished state.
